# DVSI Test Vector Manifest

DVSI's `tv-std` and `tv-rc` test vector sets are publicly downloadable
from DVSI's website but remain DVSI's copyright. We **do not redistribute
them**. Instead this directory contains SHA-256 manifests that let anyone
verify their local DVSI copy is bit-identical to the one that was
validated against during this project's development.

## Files

- `tv-std.sha256` — SHA-256 hashes of every file in DVSI's `tv-std`
  directory (standard rate configurations).
- `tv-rc.sha256` — SHA-256 hashes of every file in DVSI's `tv-rc`
  directory (rate-conversion configurations).

Both files are generated by `tools/dvsi_manifest.py --generate` and
verified by `tools/dvsi_manifest.py --verify`.

## Usage

```
# Generate a new manifest (one-time, after downloading fresh DVSI files):
python3 tools/dvsi_manifest.py --generate "/mnt/share/P25-IQ-Samples/DVSI Vectors/tv-std" \
    test_vectors/dvsi_manifest/tv-std.sha256

# Verify a local copy matches a committed manifest (regression check):
python3 tools/dvsi_manifest.py --verify "/mnt/share/P25-IQ-Samples/DVSI Vectors/tv-std" \
    test_vectors/dvsi_manifest/tv-std.sha256
```

## What the Manifest Does

- Confirms every file referenced by this project's validation work exists
  in your local DVSI copy at the same bit pattern we saw.
- Catches silent corruption of DVSI downloads.
- Makes "I tested against DVSI vectors" a reproducible claim rather than
  an assertion.

## What the Manifest Does NOT Do

- Does not contain DVSI content (only hashes and filenames). Safe to
  commit.
- Does not substitute for actually running the vectors — it only proves
  the inputs are the same.
- Does not cover DVSI software (just vectors). If we wanted to validate
  software versions too, we'd add a separate `dvsi_software.sha256`.

## License / Legal

SHA-256 hashes are cryptographic fingerprints, not derivative works of
the input content. Committing `.sha256` files is analogous to committing
a grocery receipt that mentions a brand name — factual reference, not
reproduction.

Filenames *are* reproduced here (they are part of the manifest format).
If DVSI considers their filename list proprietary, we'd need to review —
but the filenames are generic (`r0_u.pcm`, `r1_u.pcm`, etc.) and the
existence of the files is documented in DVSI's public README.
