# Related Resources & Context

## Status

This document (November 2019, original version) is **active** and part of the current Project 25 Standard. It was adopted by the P25 Steering Committee on October 10, 2019 and published by TIA on November 14, 2019. It received ANSI approval in October 2020.

An **Addendum 1** (TIA-102.BBAE-1) was published in **March 2025**, titled "Location on PTT and User Alias." This addendum extends the base document with procedures for conveying subscriber location data during push-to-talk events and user alias information over the TDMA air interface. The base document has not been superseded — the addendum supplements it.

## Practical Context

### How This Standard Applies in Real-World Systems

P25 Phase 2 TDMA is deployed in public safety radio systems across the United States and internationally. The procedures defined in this document govern how subscriber radios (portables and mobiles) and base station infrastructure interact on TDMA traffic channels. Every P25 Phase 2 trunked voice call — from a police dispatch group call to an emergency preemption — follows the call setup, traffic mode, hangtime, and termination sequences specified here.

Major infrastructure vendors implementing these procedures include Motorola Solutions (ASTRO 25 / APX series), L3Harris (VIDA / XL series), and EF Johnson / Kenwood (Viking series). All P25 Phase 2 capable subscriber radios must implement the SU-side procedures from this document to achieve interoperability.

The FCC requires P25 Phase 2 TDMA capability for new trunked systems in the 700 MHz public safety band, making this document foundational for modern public safety radio deployments.

### Key Operational Behaviors Governed by This Document

- How radios acquire and synchronize to TDMA control and voice channels
- The MAC_PTT / voice burst / MAC_END_PTT sequence for every voice transmission
- Message trunking hangtime and call continuation (CCH vs VCH methods)
- Audio preemption (console override of a talker) — forced and unforced
- Call preemption (emergency call taking over a channel) — polite and impolite transitions
- Scrambling seed computation from WACN/System ID/Color Code for co-channel interference rejection
- Dynamic LCH topology transitions (dual LCCH → single LCCH → composite)

## Open-Source Implementations

Several open-source projects implement P25 Phase 2 TDMA decoding, directly exercising the procedures and message formats defined in this document and its companions:

### SDRTrunk
- **URL**: https://github.com/DSheirer/sdrtrunk
- Cross-platform Java application for decoding P25 Phase I, Phase II (TDMA), and DMR
- Implements TDMA voice channel decoding including MAC PDU parsing, scrambling/descrambling, ISCH decoding, and call state tracking
- Active development includes TDMA control channel (LCCH) support (see issue #1304)
- P25 Phase 2 TDMA data channel support tracked in issue #2089

### OP25 (Osmocom)
- **URL**: https://github.com/osmocom/op25
- GNU Radio-based P25 decoder supporting Phase I and Phase II
- Implements TDMA demodulation, MAC layer parsing, and AMBE+2 voice decoding
- Runs on Linux with RTL-SDR or other SDR hardware

### OP25 (Boatbod fork)
- **URL**: https://github.com/boatbod/op25
- Actively maintained fork of OP25 with improvements to Phase 2 TDMA handling
- Enhanced web-based UI and configuration

### Trunk-Recorder
- **URL**: https://github.com/robotastic/trunk-recorder
- P25 trunked system recorder supporting Phase I and Phase II
- Uses OP25 decoder components for TDMA voice channel recording
