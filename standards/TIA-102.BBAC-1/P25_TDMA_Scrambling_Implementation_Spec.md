**Purpose:** Self-contained spec for implementing the TDMA scrambling/descrambling pipeline.

## 1. Overview

P25 Phase 2 TDMA uses a scrambling sequence to prevent co-channel interference between
adjacent P25 systems. The scramble sequence is generated by a 44-bit Linear Feedback Shift
Register (LFSR) seeded from system identity parameters. Each superframe (360 ms, 4320 bits)
uses its own complete scramble sequence, restarting from the seed at each superframe boundary.

Inbound and outbound directions use different seeds derived from the same system parameters.

## 2. Seed Computation

### 2.1 Inputs

Three values, all obtained from the Network Status Broadcast message:

| Parameter | Width | Description |
|-----------|-------|-------------|
| WACN_ID   | 20 bits | Wide Area Communications Network Identifier |
| System_ID | 12 bits | System Identifier |
| CC        | 12 bits | Color Code (same as NAC for Phase 1 FDMA) |

### 2.2 External Seed Calculation

```
seed_external = (WACN_ID << 24) + (System_ID << 12) + CC

Mask to 44 bits:
seed_external = seed_external & 0xFFF_FFFF_FFFF

If seed_external == 0:
    seed_external = 0xFFF_FFFF_FFFF    // (2^44 - 1)
```

The result is always in range [1, 2^44 - 1]. Zero is excluded.

### 2.3 Outbound and Inbound Seeds

```
seed_external_outbound = seed_external
seed_external_inbound  = gf2_matrix_vector_multiply(SH_MATRIX, seed_external)
```

The outbound seed is used directly. The inbound seed is derived by multiplying the
external seed by the SH matrix (Section 5) in GF(2), which is equivalent to advancing
the LFSR by 2^43 cycles.

## 3. LFSR Generator

### 3.1 Polynomial

```
G(x) = x^44 + x^40 + x^35 + x^29 + x^24 + x^10 + 1
```

Feedback tap bit positions (0-indexed from LSB): **40, 35, 29, 24, 10**

### 3.2 Galois LFSR — As Described in TIA Spec

> **ERRATA:** The TIA spec labels this "External LFSR (Fibonacci form)" but the
> algorithm (XOR taps into state when output=1) is actually the **Galois** form.
> This was verified against SDRTrunk, OP25, and the mathematical definition.
> See Section 3.3 for the Fibonacci form actually used by implementations.

The Galois LFSR XORs the tap positions into the shifted state when the output bit is 1.
Output is taken from the MSB (bit 43).

```
TAPS = (1 << 40) | (1 << 35) | (1 << 29) | (1 << 24) | (1 << 10)
     = 0x108_2100_0400

MASK_44 = 0xFFF_FFFF_FFFF

function galois_lfsr_step(state) -> (new_state, output_bit):
    output_bit = (state >> 43) & 1
    new_state = (state << 1) & MASK_44
    if output_bit == 1:
        new_state = new_state XOR TAPS
    return (new_state, output_bit)
```

> **ERRATA:** The original extraction had TAPS = 0x102_2100_0400, which has bit 33
> set instead of bit 35. The correct value for the polynomial
> G(x) = x^44 + x^40 + x^35 + x^29 + x^24 + x^10 + 1 is **0x108_2100_0400**.
> Verified: (1<<40)=0x100_0000_0000, (1<<35)=0x8_0000_0000,
> (1<<29)=0x2000_0000, (1<<24)=0x100_0000, (1<<10)=0x400.
> Sum = 0x108_2100_0400.

### 3.3 Fibonacci LFSR — Used by SDRTrunk/OP25 (Recommended)

The Fibonacci form is what SDRTrunk and OP25 actually implement. It XORs the tap
positions to produce a single feedback bit that enters at position 0.

This uses the **reciprocal polynomial**:
```
G*(x) = x^44 + x^34 + x^20 + x^15 + x^9 + x^4 + 1
```

Tap positions (0-indexed): **43, 33, 19, 14, 8, 3**

```
function fibonacci_lfsr_step(state) -> (new_state, output_bit):
    output_bit = (state >> 43) & 1
    feedback = ((state >> 43) ^ (state >> 33) ^ (state >> 19)
              ^ (state >> 14) ^ (state >> 8) ^ (state >> 3)) & 1
    new_state = ((state << 1) | feedback) & MASK_44
    return (new_state, output_bit)
```

The Fibonacci form with the reciprocal polynomial and direct seed produces the
same output sequence as the Galois form with the original polynomial and
M-matrix-converted seed. Both are correct; the Fibonacci form is simpler
because it avoids the M-matrix conversion entirely.

### 3.4 Galois-to-Fibonacci Equivalence

The TIA polynomial G(x) in Galois form with M-matrix seed conversion produces
the same sequence as the reciprocal polynomial G*(x) in Fibonacci form with
direct seed. SDRTrunk and OP25 use the latter.

```
// These produce identical output sequences:
galois_lfsr(G_poly, M_MATRIX * seed)  ==  fibonacci_lfsr(G_reciprocal, seed)
```

### 3.5 Generating the Scramble Sequence

```
function generate_scramble_sequence(seed, length=4320) -> bit_array:
    state = seed
    bits = new array[length]
    for i in 0..length-1:
        (state, bit) = lfsr_step(state)
        bits[i] = bit
    return bits
```

as shown; do NOT pre-transpose SH_MATRIX before use.

## 6. GF(2) Matrix-Vector Multiplication

This operation is used for seed derivation (SH matrix) and optionally for LFSR state conversion (M matrix).

Each matrix row is AND'd with the input vector, then the popcount (number of set bits) modulo 2 gives the output bit for that row position.

```
function gf2_matrix_vector_multiply(matrix[44], vector) -> result:
    result = 0
    for i in 0..43:
        product = matrix[i] AND vector
        bit = popcount(product) MOD 2      // or: parity(product)
        if bit == 1:
            result = result OR (1 << (43 - i))
    return result
```

**C note:** Use `__builtin_parityll(product)` (GCC/Clang) or count set bits manually. The parity check for a 64-bit word is `__builtin_parityll(matrix[i] & vector)`.

## 7. Superframe Timing and Offsets

A superframe = 360 ms = 12 slots (numbered 0-11) = 4320 bits per channel direction.
Each slot = 30 ms = 360 bits.

The scramble sequence of 4320 bits covers one full superframe and restarts at each
superframe boundary.

### 7.1 Outbound

The outbound scramble sequence runs continuously from offset 0 to 4319 across the superframe.
Note: the outbound superframe boundary falls mid-way through an ISCH from the previous slot.

### 7.2 Inbound (VCH0 and VCH1)

The inbound scramble sequence uses the inbound seed and maps to slot boundaries:

| Slot | VCH0 Start | VCH0 End | VCH1 Start | VCH1 End |
|------|-----------|---------|-----------|---------|
| 0    |           |         | 0         | 359     |
| 1    | 360       | 719     |           |         |
| 2    |           |         | 720       | 1079    |
| 3    | 1080      | 1439    |           |         |
| 4    |           |         | 1440      | 1799    |
| 5    | 1800      | 2159    |           |         |
| 6    |           |         | 2160      | 2519    |
| 7    | 2520      | 2879    |           |         |
| 8    |           |         | 2880      | 3239    |
| 9    | 3240      | 3599    |           |         |
| 10   |           |         | 3960      | 4319    |
| 11   | 3600      | 3959    |           |         |

VCH0 occupies even-numbered slots; VCH1 occupies odd-numbered slots.

## 8. What Gets Scrambled

### 8.1 Always Scrambled
- Voice frames and ESS fields in all 2V and 4V bursts (inbound and outbound)
- Signaling bursts on FACCH and SACCH (with exceptions below)

### 8.2 Never Scrambled
- Ramp/Guard time symbols
- Pilot sequences (P1, P2)
- DUID fields
- ISCH fields
- Sync sequences
- Any FDMA channel content (control or traffic)
- Signaling bursts containing a **Network Status Broadcast** MAC message
- Signaling bursts containing a **MAC_END_PDU**

### 8.3 Scrambling Indication
The DUID bits (Table 5-3 in the base spec) indicate whether SACCH/FACCH bursts are scrambled.

### 8.4 Application Point
- **Transmitter:** Scramble after encoding (including FEC) but before modulation
- **Receiver:** Descramble after demodulation but before FEC decoding or any other decoding

## 9. Complete Pipeline Pseudocode

```c
#include <stdint.h>
#include <stdbool.h>

#define MASK_44     UINT64_C(0xFFFFFFFFFFF)   /* (1<<44) - 1 */
#define TAPS_GALOIS UINT64_C(0x10821000400)   /* (1<<40)|(1<<35)|(1<<29)|(1<<24)|(1<<10) */

/* === INITIALIZATION (once per system or when Network Status Broadcast changes) ===
 * Computes the outbound and inbound seeds from system identity parameters.
 * All three inputs come from the Network Status Broadcast MAC message.
 */
void init_scrambler(uint32_t wacn_id, uint16_t system_id, uint16_t color_code,
                    uint64_t *outbound_seed, uint64_t *inbound_seed)
{
    uint64_t seed = ((uint64_t)wacn_id << 24)
                  + ((uint64_t)system_id << 12)
                  + (uint64_t)color_code;
    seed &= MASK_44;
    if (seed == 0)
        seed = MASK_44;   /* exclude zero per spec */

    *outbound_seed = seed;
    *inbound_seed  = gf2_matrix_vector_multiply(SH_MATRIX, seed);
}


/* === PER-SUPERFRAME (called at start of each 360 ms superframe) ===
 * Generates 4320 scramble bits by running the Galois LFSR for 4320 steps.
 * Uses the Galois (internal-feedback) form; see §3.2 for the Fibonacci
 * equivalent that avoids the M-matrix seed conversion.
 */
void generate_superframe_scramble(uint64_t seed, uint8_t sequence[4320])
{
    uint64_t state = seed;
    for (int i = 0; i < 4320; i++) {
        uint64_t output = (state >> 43) & 1;
        state = (state << 1) & MASK_44;
        if (output)
            state ^= TAPS_GALOIS;
        sequence[i] = (uint8_t)output;
    }
}


/* === PER-BURST (apply scrambling to the scrambable bits of one burst) ===
 * XORs each scrambable burst bit with the corresponding scramble sequence bit.
 * Non-scrambable fields (ramp/guard, pilot, DUID, ISCH, sync) are skipped.
 * offset = first bit index in the 4320-bit superframe sequence for this slot.
 * scramble_map = array of (burst_bit_index, sequence_offset) pairs for this
 *                burst type; derived from Annex E tables in
 *                P25_TDMA_Annex_E_Burst_Bit_Tables.md.
 */
void scramble_burst(uint8_t *burst_bits, const uint8_t *scramble_seq,
                    int offset, const int (*scramble_map)[2], int map_len)
{
    for (int k = 0; k < map_len; k++) {
        int bit_idx      = scramble_map[k][0];
        int seq_idx      = offset + scramble_map[k][1];
        burst_bits[bit_idx] ^= scramble_seq[seq_idx];
    }
}
```

## 12. C Implementation Notes

- Store seeds and LFSR state as `uint64_t` (44 bits used; upper 20 bits zero).
- Matrix rows stored as `uint64_t[44]`.
- Parity for GF(2) matrix-vector multiply: `__builtin_parityll(matrix[i] & vector)`
  (GCC/Clang). On MSVC use `_mm_popcnt_u64(x) & 1`. Alternatively, fold a
  `uint64_t` with six XOR-shift steps to get parity without a popcount intrinsic.
- The LFSR inner loop can be made branch-free:
  ```c
  uint64_t output = (state >> 43) & 1;
  state = ((state << 1) & MASK_44) ^ (TAPS_GALOIS * output);
  /* TAPS_GALOIS * 0 = 0, TAPS_GALOIS * 1 = TAPS_GALOIS */
  ```
- For bulk generation (4320 bits per superframe), unroll the loop and pack
  output bits into `uint64_t` words (64 bits per iteration) if throughput matters.
- Per-burst scramble application: a plain `uint8_t[4320]` array (one bit per
  byte) is simplest; index directly using the Annex E offset tables.
