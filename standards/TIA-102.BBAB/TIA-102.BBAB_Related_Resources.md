# TIA-102.BBAB — Related Resources & Context

## Status

This document was published in **July 2009** and has been **reaffirmed** (confirmed as still current without revision). As of the most recent P25 Steering Committee approved standards lists (2018–2019), it remains an **active, approved P25 standard** with no superseding document. It has ANSI approval concurrent with its TIA publication date.

The document has not been revised since initial publication. The companion MAC layer document (TIA-102.BBAC) received an addendum (TIA-102.BBAC-1, February 2013) and the MAC layer messages document (TIA-102.BBAD) was published in August 2017, but the physical layer specification itself has remained stable.

## Practical Context

### How This Standard Applies in Real-World Systems

P25 Phase 2 TDMA is deployed in large metropolitan and statewide public safety radio systems where spectrum efficiency is critical. By placing two voice paths on a single 12.5 kHz channel, agencies effectively double their channel capacity without additional spectrum. Major deployments include state patrol systems, large urban police/fire networks, and federal agency systems.

The asymmetric modulation design is a key practical feature:
- **Subscriber units (portables/mobiles)** use H-CPM, a constant-envelope modulation that works with efficient Class C amplifiers — critical for battery life in handheld radios.
- **Base stations/repeaters** use H-DQPSK, which requires linear amplifiers but is feasible in fixed infrastructure where power efficiency is less constrained.

Phase 2 systems are backward-compatible with Phase 1 subscribers. A Phase 2 RFSS can dynamically assign calls to FDMA or TDMA channels depending on the capabilities of the participating radios (Motorola calls this "Dynamic Dual Mode" or DDM). The Phase 1 FDMA control channel is retained, and Phase 2 TDMA is used only for traffic (voice) channels in trunked operation.

Major equipment vendors implementing this standard include **Motorola Solutions** (ASTRO 25 / APX series), **L3Harris** (formerly Harris — VIDA / XL series), **Tait Communications** (TP/TM9400 series), and **EF Johnson** (now part of Kenwood/JVCKENWOOD — Viking VP series).

## Open-Source Implementations

Several open-source projects implement or interact with the physical layer protocols defined in this document:

### OP25 (Osmocom)
- **Repository (boatbod fork)**: https://github.com/boatbod/op25
- **Upstream (Osmocom)**: https://gitea.osmocom.org/op25/op25
- GNU Radio-based P25 decoder supporting Phase 1 and Phase 2 TDMA. Includes TDMA demodulation code implementing the H-DQPSK differential phase detection described in this standard. Supports trunking follow on both Phase 1 and Phase 2 systems. The `tdma` directory contains the Phase 2 slot processing logic.

### SDRTrunk
- **Repository**: https://github.com/DSheirer/sdrtrunk
- **Releases**: https://github.com/DSheirer/sdrtrunk/releases
- Cross-platform Java SDR application for monitoring P25 (and DMR) trunked systems. Implements P25 Phase 2 TDMA voice channel decoding including ISCH parsing, AMBE+2 vocoder integration, and H-DQPSK demodulation. Active development on TDMA control channel support (see issue #1304).

### Trunk-Recorder
- **Repository**: https://github.com/TrunkRecorder/trunk-recorder
- Records calls from P25 trunked systems using SDR hardware. Supports Phase 2 TDMA traffic channels via embedded OP25 decoder components. Handles the TDMA slot identification and frequency table parsing needed to follow Phase 2 voice channel grants.

### JMBE (Java Multi-Band Excitation)
- **Repository**: https://github.com/DSheirer/jmbe
- Java implementation of the AMBE+2 vocoder used for P25 Phase 2 audio decoding. Required by SDRTrunk for Phase 2 voice playback. Implements the half-rate vocoder codec referenced by this standard (via TIA-102.BABA-1).
